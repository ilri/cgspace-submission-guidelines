CGSpace Submission Guidelines

This page provides guidelines for programmatic content submission to CGSpace.

In this documentation we detail mandatory metadata, optional metadata, example controlled vocabularies, and provide technical information about CGSpace APIs.

Required Metadata

dcterms.accessRights dc.contributor.author dcterms.bibliographicCitation dcterms.issued dcterms.type dc.title dcterms.license

dcterms.accessRights
Required
Definition Accessibility status (required)
Comments Information about who can access the resource or an indication of its security status. May include information regarding access or restrictions based on privacy, security, or other policies. In the CGSpace context this is usually “Open Access” or “Limited Access”.
Vocabulary dcterms-accessRights.txt

dc.contributor.author
Required
Definition Authors (required)
Comments Click below to see a pre-populated list of authors, or enter the names of individual or corporate authors of this item (enter each individually).
Vocabulary dc-contributor-author.txt

dcterms.bibliographicCitation
Required
Definition Citation (required)
Comments Recommended practice is to include sufficient bibliographic detail to identify the resource as unambiguously as possible.

dcterms.issued
Required
Definition Date of publication (required)
Comments Date of formal issuance (e.g., publication) of the resource.

dcterms.type
Required
Definition Item type (required)
Comments The nature or genre of the resource.
Vocabulary dcterms-type.txt

dc.title
Required
Definition Title (required)
Comments Title statement/title proper.

dcterms.license
Required
Definition Usage rights (required)
Comments A legal document giving official permission to do something with the resource. Prefer using SPDX License Identifiers, but there are some other generic suggestions here: https://ilri.github.io/cgspace-submission-guidelines/dcterms-license/dcterms-license.txt
Vocabulary dcterms-license.txt

cg.isbn
Definition ISBN
Comments The ISBN for the item, for example: 978-3-16-148410-0 (the dashes are not necessary, as long as the ISBN is valid). If the item has multiple ISBNs please use multiple fields. Do not enter any other information here.

cg.issn
Definition ISSN
Comments The ISSN for the serial publication where this item appears, for example: 2049-3630. If the item has multiple ISSNs, for example print and online, please use multiple fields. Do not enter any other information like “online” here.

cg.identifier.iwmilibrary
Definition IWMI Library ID
Comments IWMI Library identifier, for example: H049940.

cg.issue
Definition Journal issue
Comments Enter the issue number of a journal, magazine, or report. For example, if published in PLoS ONE 16(1), the issue number is: 1.

cg.isijournal
Definition Journal status
Comments Enter “ISI Journal” if the journal is ISI. Do not use this field unless the journal is ISI.
Vocabulary cg-isijournal.txt
